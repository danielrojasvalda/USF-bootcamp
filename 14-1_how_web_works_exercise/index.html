# ****How Web Works Exercise****

## **Part One: Solidify Terminology**

In your own terms, define the following terms:

  • What is HTTP?

    • Hypertext Transfer Protocol. It is the foundation of data communication on the web and defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.

  • What is a URL?

    • Uniform Resource Locator. It is the address used to identify resources on the web, such as web pages, images, videos, the domain name, the path to the resource.

  • What is DNS?

    • Domain Name System. It is a system that translates human-readable domain names (e.g., "example.com") into IP addresses (e.g., "192.0.2.1"). DNS enables us to access websites using domain names instead of remembering complex IP addresses.

  • What is a query string?

    • A query string is a part of a URL that follows the question mark (?). It is used to send data to a web server as key-value pairs. For example, in the URL "https://example.com/search?q=keyword", the query string is "q=keyword" where "q" is the key and "keyword" is the value.

  • What are two HTTP verbs and how are they different?

    • HTTP verbs, also known as HTTP methods, are used to indicate the desired action to be performed on a resource. Two common HTTP verbs are:

    • GET: Used to retrieve data from a server.
    • POST: Used to submit data to be processed to a specified resource.

  • What is an HTTP request?

    • An HTTP request is a message sent by a client (usually a web browser) to a web server to request a specific resource.

  • What is an HTTP response?

    • An HTTP response is a message sent by a web server back to the client in response to an HTTP request.

  • What is an HTTP header? Give a couple examples of request and response headers you have seen.

    • HTTP headers are the name or value pairs that are displayed in the request and response messages
    Request Header:
    User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/94.0.4606.81 Safari/537.36
    Response Header:
    Content-Type: text/html; charset=UTF-8

  • What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?

    • When typing "http://somesite.com/some/page.html" into a browser, the following processes happen:

    1. The browser checks the DNS to resolve the domain name "somesite.com" into an IP address.
    2. The browser establishes a TCP connection to the web server at the resolved IP address.
    3. An HTTP GET request is sent to the server for the resource "some/page.html".
    4. The server processes the request and sends back an HTTP response with the requested HTML page.
    5. The browser receives the response and renders the HTML page, displaying it to the user.
